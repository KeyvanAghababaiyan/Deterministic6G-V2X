Deterministic 6G-V2X Systems

Towards reliable, bounded-latency communication–computing–control for Connected and Automated Vehicles.

The Deterministic6G-V2X project explores how future 6G networks can guarantee predictable end-to-end latency for safety-critical V2X applications. By integrating communication, edge/cloud computing, and control in a single deterministic architecture, the project aims to support cooperative and automated driving with strict timing constraints.

Project summary

Connected and Automated Vehicles (CAVs) generate massive data streams from sensors such as lidar, radar, and cameras, which must be processed in real time and combined with information received via V2X links. Current on-board processing is often insufficient to handle these loads with strict latency constraints, especially in dense traffic scenarios.

Future 6G systems will extend 5G's Mobile Edge Computing capabilities, embedding AI and advanced networking across the local–edge–cloud continuum. Deterministic6G-V2X studies how to allocate computation across this continuum and how to design queueing and scheduling mechanisms so that communication, computing, and control remain within bounded delays.

Key objectives

  • Model end-to-end latency including communication, computation, and control loops for V2X services.
  • Design deterministic mechanisms for task offloading across local, edge, and cloud resources.
  • Evaluate reliability and deadline-miss probabilities under realistic traffic and network conditions.
  • Provide guidelines for deterministic 6G-V2X architectures supporting cooperative driving.
